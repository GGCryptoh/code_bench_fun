Usage

Two ways to run a bench: the local CLI runner (Claude models ride your Claude Code subscription) or the One-Shot Bench page in the browser (everything via OpenRouter, bring your own key).

Run a bench from the terminal

One command builds the same game with every model in parallel, counts time / tokens / cost / turns, has Opus 4.8 judge each build, and writes everything into the gallery:

node runner/bench.mjs \
  --id cars-vs-canyon \
  --title "Cars vs Canyon" \
  --kind simulation \
  --models fable-5,glm-5.2,gpt-5.5,opus-4.8,grok-4.20,gemini-3.5-flash \
  --prompt "A 2D side-view physics simulation: a red car and a blue car race in \
from the left, hit launch ramps and jump across a deep canyon side by side. \
Suspension compression and bounce on landing, kicked-up dust and debris \
particles, parallax sky with drifting clouds and a warm sun. After each jump \
the cars reset and run again with slightly different speeds, forever. Fixed \
camera, no user input, pure autoplaying simulation."

# then preview locally…
python3 -m http.server 8619        # → http://127.0.0.1:8619/run.html?id=cars-vs-canyon

# …and publish
git add -A && git commit -m "cars vs canyon run" && git push

Flags

--idkebab-case run id — becomes the URL (run.html?id=…) and folder name
--titledisplay title in the gallery
--kindsimulation (autoplays, loops, no input — best for reels) or interactive (playable)
--modelscomma-separated model keys (see below); panel order = this order, reel mode shows the first 4 successful
--prompt / --prompt-filethe one-shot game prompt, inline or from a file
--judge-providerclaude-cli (default, subscription) or openrouter
--force-openrouterrun Claude models through OpenRouter too (real API cost, cleaner apples-to-apples input tokens)

Model keys

OpenRouter key

Non-Claude models need an OpenRouter key, resolved in this order:

1. env varOPENROUTER_API_KEY=sk-or-… node runner/bench.mjs …
2. .env.localfile at the repo root with OPENROUTER_API_KEY=sk-or-… (gitignored; the bench page also picks it up on localhost)
3. macOS keychainservice OPENROUTER_API_KEY

Claude models (Fable 5, Opus 4.8, Sonnet 5, Haiku 4.5) run through the local claude CLI on your Claude subscription — no API key needed.

OpenAI models (GPT 5.5, GPT 5.1) go direct to api.openai.com when an OPENAI_API_KEY is available (same resolution order: env → .env.local → keychain service OPENAI_KEY) — exact first-party billing, no OpenRouter spend-limit preauth. No key → they fall back to OpenRouter. --force-openrouter overrides.

Import a browser run

Built something on the One-Shot Bench page? Download the run bundle, then:

node runner/import.mjs cbf-run-my-game-20260705.json
git add -A && git commit -m "import run" && git push

Recording for social

Open a run → Reel mode (or add ?reel=1). The grid fits the viewport, counters race at true relative speed and loop every ~24s. Tune with ?replay=12 (race seconds) and ?loop=1 (loop outside reel mode too). ESC exits. Costs are real — big models can be $1+ per game.
